# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 6.x | ✅ |
| 5.x | ✅ (critical fixes only) |
| < 5.0 | ❌ |

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in Max Super Prompt or any part of the Max Skill Framework:

1. **DO NOT** open a public issue
2. Email the maintainer at **security@max-skill.dev** or DM **@Jenzo0** on GitHub
3. Provide a detailed description of the vulnerability
4. Include steps to reproduce if applicable
5. If possible, suggest a fix

You'll receive an acknowledgment within 48 hours. We'll keep you informed of the progress toward a fix and release.

## Scope

This policy applies to:

- `max-super-prompt/SKILL.md` and all reference files
- `lite/SKILL.md`
- Scripts in `references/scripts/`
- Documentation and configuration files

Out of scope:

- Third-party prompts derived from Max
- User-specific customizations and forks
- Platform vulnerabilities (Hermes, ChatGPT, Claude, etc.)

## Disclosure

We follow coordinated disclosure: 90-day window from report to public disclosure, or sooner if a fix is released.
